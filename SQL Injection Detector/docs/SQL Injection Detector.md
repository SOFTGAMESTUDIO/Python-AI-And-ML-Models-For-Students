## 🏗️ System Architecture

The Soft Game Studio SQL Injection Detector is built on a modular architecture separating the user interface from the core scanning engine. This design ensures scalability, ease of maintenance, and reliable vulnerability detection.

### 1. Frontend Interface (`app.py`)
The user interface is powered by a Flask web server that handles client requests, initializes the scanning processes, and serves the vulnerability reports dynamically.

- **Routing & API**: Defines RESTful endpoints such as `/api/scan` for starting asynchronous scans and `/api/reports` for retrieving historical scan data.
- **Report Management**: Manages the storage, retrieval, and JSON serialization of vulnerability reports generated by the backend engine, allowing users to view or download findings.

### 2. Core Scanning Engine (`backend/scanner.py`)
The `SQLiScanner` class is the heart of the application, orchestrating the end-to-end testing process against targeted URLs.

- **Payload Injection (`build_test_url`)**: Parses the target URL, extracts existing query parameters, systematically injects malicious payloads into specific parameters, and reconstructs the URL for testing.
- **Parameter Testing (`test_parameter`)**: Sends the injected HTTP requests and analyzes the responses. It employs three distinct detection strategies to maximize coverage:
  - **Error-Based Detection**: Uses Regular Expressions (Regex) to scan the response body for common database error messages (e.g., MySQL syntax errors, Oracle warnings, PostgreSQL exceptions).
  - **Time-Based Detection**: Accurately measures the response time of the server. If a specific payload (like `SLEEP(5)`) causes a significant and measurable delay, a time-based vulnerability is flagged.
  - **Boolean-Based Detection**: Compares the character length of the response from the injected request against the original baseline response. Significant variations indicate that the payload successfully altered the underlying query logic.
- **Reporting (`generate_report`)**: Aggregates all detected findings, categorizes them by severity (CRITICAL for Error-based, HIGH for Time-based, MEDIUM for Boolean-based), and outputs a structured JSON report to the file system.

### 3. Attack Surface Expansion (`backend/crawler.py` & Deep Scan)
When the "Deep Scan" feature is enabled in the UI, the engine utilizes the `BeautifulSoup` library to parse the HTML structure of the target URL, extracting all internal `<a>` links. It then recursively applies the core scanning logic to these newly discovered pages and their parameters, significantly increasing the thoroughness of the security audit.

### 4. Payloads & Detectors (`backend/payloads.py` & `backend/detectors.py`)
- **Payloads**: Contains a specialized, curated list of SQL injection vectors designed to trigger various database behaviors across different backend database management systems (DBMS).
- **Detectors**: Extensible Python modules that abstract the complex logic of analyzing HTTP responses, allowing developers to easily introduce new, custom detection heuristics in future updates.

## ⚙️ Security and Ethical Considerations

This application is strictly designed for educational purposes, authorized security auditing, and penetration testing on systems you own or have explicit permission to test. Running automated vulnerability scanners against targets without authorization is illegal and unethical.
